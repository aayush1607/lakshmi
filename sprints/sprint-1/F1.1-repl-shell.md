# F1.1 — REPL Shell

## One-liner
A terminal REPL that greets the user, accepts commands, and keeps history.

## User story
> As a new Lakshmi user, I want to launch `lakshmi` and see a prompt I can type into, so I know the tool is alive and I can start exploring.

## Expected behaviour (PM view)
- User runs `lakshmi` in their terminal.
- A welcome banner appears (product name, version, tagline, one-line help hint).
- A prompt appears: `🪷 › `
- User can type text and press Enter — the input echoes back (even if the command isn't implemented yet, it is acknowledged: *"command not recognised — try /help"*).
- `/help` lists available commands (just `/help` and `/exit` for now).
- `/exit` or `Ctrl-D` cleanly closes the shell.
- Up-arrow recalls previous commands (history persists across sessions in `~/.lakshmi/history`).
- Ctrl-C cancels the current input line without exiting the shell.

## Acceptance criteria
1. Launching `lakshmi` shows the banner and prompt within 500 ms.
2. Typing gibberish gets a polite "unknown command" response, not a crash.
3. History survives restart.
4. `/exit` returns the user to their normal shell with exit code 0.
5. No network calls are made during startup.

## Out of scope
- Any real command logic (portfolio, ta, fa, ask).
- Authentication.
- Styled/coloured output beyond the banner (nice-to-have, not required).

## Dependencies
None — this is the foundation everything else plugs into.
