# F1.2 — Zerodha Login

## One-liner
A one-command flow to connect the user's Zerodha account and keep the session alive locally.

## User story
> As an Indian retail investor, I want to link my Zerodha account once and have Lakshmi remember it, so I don't have to log in every time I want to see my holdings.

## Expected behaviour (PM view)
- Inside the shell, user types `/login zerodha` (or from CLI: `lakshmi login`).
- Lakshmi opens the Zerodha OAuth/Kite login page in the user's default browser.
- User completes login on Zerodha's site.
- On success, Lakshmi shows:
  ```
  ✓ Connected as AAYUSH (ZK1234)
     Session valid until 3:30 AM tomorrow.
  ```
- Session token is stored locally (encrypted at rest, in `~/.lakshmi/`).
- `/login status` shows whether a session is active and when it expires.
- `/logout` revokes the local session.
- If the session is expired when the user runs a portfolio command, Lakshmi prompts them to `/login` again — no silent failures.

## Acceptance criteria
1. A fresh user can go from `lakshmi login` to "Connected as …" in under 60 seconds.
2. Token is **never** printed to stdout or logs.
3. Token is not stored in plaintext.
4. Closing and reopening the shell keeps the session active until its natural expiry.
5. `/login status` accurately reflects connected/expired/disconnected states.
6. A clear error appears if the user cancels the browser login (no hang).

## Out of scope
- Any broker other than Zerodha.
- Reading holdings (that's F1.3).
- Refreshing tokens automatically — user re-logs in at expiry.

## Dependencies
- F1.1 (REPL) for the in-shell command.

## Demoable in
A terminal recording: `/login zerodha` → browser opens → login → back to terminal with success. Then `/login status` shows connected. Close terminal, reopen, `/login status` still shows connected.
