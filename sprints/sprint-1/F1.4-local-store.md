# F1.4 — Local Data Store

## One-liner
Everything Lakshmi fetches is cached locally so the app is fast, offline-tolerant, and auditable.

## User story
> As a user who cares about speed and privacy, I want Lakshmi to remember what it already fetched so I'm not hitting APIs every second — and so I can see what data the app is using.

## Expected behaviour (PM view)
- All holdings, quotes, fundamentals, and news fetched by Lakshmi are written to a local store in `~/.lakshmi/data/`.
- Any command that has "recent enough" cached data uses the cache by default:
  - Holdings: fresh if < 2 min old during market hours, < 1 day off-hours.
  - Quotes: fresh if < 60 s during market hours.
  - Fundamentals: fresh if < 24 h.
- User can force a live fetch with `--fresh`, e.g. `/portfolio --fresh`.
- `/cache status` shows:
  ```
  Holdings:     12 KB   last refresh 2 min ago
  Quotes:      860 KB   last refresh 40 s ago
  Fundamentals: 4.2 MB  last refresh 18 h ago
  Total:        5.1 MB
  ```
- `/cache clear` wipes local data (with confirmation prompt).
- Cache is single-user and lives entirely on the user's machine — nothing is ever uploaded.

## Acceptance criteria
1. A cached `/portfolio` renders in < 200 ms.
2. `--fresh` always bypasses cache and re-hits the source.
3. `/cache status` numbers match actual disk usage within 10%.
4. `/cache clear` removes all data files and is reversible only by re-fetching.
5. Store survives restarts and OS reboots.

## Out of scope
- Multi-user / shared cache.
- Cloud sync.
- Encryption of cached market data (login tokens stay encrypted per F1.2; public market data does not need to be).

## Dependencies
- F1.1 (REPL) for the `/cache` command.

## Demoable in
Run `/portfolio`, then `/cache status` shows holdings. Run `/portfolio` again — sub-second. Run `/portfolio --fresh` — hits network. Run `/cache clear`, confirm, then `/cache status` shows empty.
