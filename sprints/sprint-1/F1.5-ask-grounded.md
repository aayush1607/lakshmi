# F1.5 — Grounded `ask` Command

## One-liner
User can ask any question in natural language and get an answer that cites every source it used.

## User story
> As an investor, I want to ask Lakshmi things like "what's my IT exposure?" or "summarise TCS's last quarter" and get an answer I can trust because I can see exactly where the facts came from.

## Expected behaviour (PM view)
- User types a free-form question, either via `ask …` or just typing without a slash:
  ```
  🪷 › what's my exposure to IT?
  ```
- Lakshmi shows a short plan before answering (so the user sees what it's about to do):
  ```
  ⟳ plan: read holdings · classify sectors · summarise
  ```
- Answer arrives with:
  - A **direct response** (1–3 sentences).
  - A **sources** block listing every source actually used — with tier label (Tier 1 / 2 / 3).
  - A **confidence** line based on source count, conflicts, and recency.
- If Lakshmi doesn't have a grounded source for a claim, it says **"I don't have a source for that"** rather than guessing.
- User can run `/why` after any answer to expand the reasoning trace (which tools were called, what they returned).
- Works offline for anything the cache already has; gracefully degrades otherwise.

## Acceptance criteria
1. Every factual claim in an answer maps to a listed source.
2. Asking a question with no available grounded source returns a refusal, not a made-up answer.
3. `/why` reveals the exact tool calls and their inputs/outputs.
4. A user can ask three follow-up questions in one session and each answer correctly uses prior context (e.g. "what about energy?" after the IT question).
5. Average answer time under 10 seconds for simple portfolio questions.

## Out of scope
- Verdict formatting (F1.6 wraps this output).
- Charts / tables rendered as part of the answer (Sprint 2 adds those).
- Voice input.
- Cross-session memory of past questions.

## Dependencies
- F1.3 (Portfolio read) — so questions about the user's holdings have grounding.
- F1.4 (Local store) — for fast repeated queries.

## Demoable in
A recording of 3–4 diverse questions (one portfolio, one stock, one "I don't know" refusal) with their sources and a `/why` expansion.
