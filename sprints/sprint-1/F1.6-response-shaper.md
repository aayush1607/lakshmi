# F1.6 — Universal Verdict Format (Response Shaper)

## One-liner
Every answer Lakshmi gives follows the same shape: **Verdict → Why → Detail → Sources → Next actions**.

## User story
> As a user who's time-poor, I want to get the bottom line in one glance and drill down only if I care — so Lakshmi never makes me read five paragraphs to find "yes" or "no".

## Expected behaviour (PM view)
- Every answer — from `ask`, `/portfolio`, `/ta`, `/fa`, etc. — follows the same visual grammar:
  ```
  ━━━ VERDICT ━━━
  🟢 / 🟡 / 🔴  <one-line call>

  ━━━ WHY ━━━
  • bullet 1
  • bullet 2
  • bullet 3

  ━━━ DETAIL ━━━
  (tables, numbers, expanded prose — hidden behind `more` if long)

  ━━━ SOURCES ━━━
  [1] source-name — what it provided   (Tier 1/2/3)
  [2] …

  ━━━ CONFIDENCE: NN% ━━━
  Based on X sources · Y conflicts · recency: <age>

  📎 Next: <suggested follow-up commands>
  ```
- **Verdict colour rules (consistent everywhere):**
  - 🟢 Green — clear positive / safe / do-it.
  - 🟡 Yellow — mixed / lean one way / conditions apply.
  - 🔴 Red — clear negative / avoid / don't do it.
- "DETAIL" section collapses by default when > 20 lines. User types `more` to expand.
- "Next" suggestions are real, runnable commands.
- When a question is purely informational ("what is ROCE?"), the Verdict slot is replaced by a one-line **answer**, same shape.

## Acceptance criteria
1. No command in Sprint 1 ships an answer that skips this format.
2. A user can find the verdict in under 1 second of visual scanning.
3. Every "WHY" bullet ties back to at least one item in SOURCES.
4. "Next" suggestions always work when executed.
5. The same question asked twice in a row produces the same visual structure (determinism of layout).

## Out of scope
- Personalisation of format (all users see the same shape in v1).
- Export to PDF / markdown — v1.5.
- Multiple languages.

## Dependencies
- F1.5 (ask) — this feature is the presentation layer on top of it.
- F1.3 (portfolio) — re-skins its output into this format.

## Demoable in
Side-by-side: run `ask`, `/portfolio`, and an unknown command — all three responses follow the same grammar.
