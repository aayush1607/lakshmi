# F2.1 — NSE OHLCV Data Pipeline

## One-liner


## User story
> As a user who wants fast technical analysis without paying for data, I want Lakshmi to have a clean local price history so charts and indicators work instantly and offline.

## Expected behaviour (PM view)
- On first run (or via `/data sync`), Lakshmi downloads NSE historical daily bars for all listed equities from the trailing 10 years.
- Progress bar shows: `Downloading NSE history … 2,145 / 2,180 symbols · 9.2 min remaining`.
- After the first sync, daily incremental updates are pulled automatically after market close (or at next launch if the user was offline).
- `/data status` shows:
  ```
  Universe:      2,180 NSE equities
  History:       10 years daily (2016-04-17 → 2026-04-16)
  Last update:   16 Apr 2026, 18:42 IST
  Disk:          ~1.8 GB
  ```
- `/data refresh <ticker>` forces a targeted update for one symbol.
- Corporate actions (splits, bonuses, dividends, mergers) are applied so price series are adjusted.
- A **ticker ontology** maps NSE symbol ↔ BSE symbol ↔ ISIN ↔ Zerodha tradingsymbol ↔ Screener slug, surviving renames and re-listings.
- Source is always NSE bhavcopy (Tier 1) — labelled as such whenever used.

## 🏰 Moat contribution
This is one of the two load-bearing features of the product's defensibility. LLMs can't reason over data they don't have, and Indian market data is fragmented and inconsistent. The value isn't "we have prices" — it's **adjusted, point-in-time, cross-identifier-reconciled** prices. Spend the time to get corporate actions and the ticker ontology right; everything downstream (TA, backtests, rebalance, tax) inherits this correctness or this error.

## Data strategy (v1)
**No paid APIs. No partnerships. Everything below is free and Tier-1.**

| Dataset | Primary source | How we fetch | Fallback / verify |
|---|---|---|---|
| Daily OHLCV | **NSE bhavcopy** (archives.nseindia.com) daily ZIP/CSV | Polite fetcher: 1 req/sec, realistic user-agent, cached forever once pulled | BSE bhavcopy for BSE-only names |
| Corporate actions | **NSE corporate announcements** + **BSE corporate actions feed** | Daily poll; parse splits, bonuses, dividends, mergers, name changes | Cross-check adjustments against Zerodha Kite historical API using the dev's own token (never shipped) |
| Ticker ontology | NSE equity master + BSE equity master + ISIN directory | One-time download, refreshed weekly | Zerodha instruments CSV (public) as tie-breaker |
| Delisting / suspension | NSE circulars | Weekly poll | — |

**Rules of the road:**
- Respect `robots.txt`; cap concurrency at 2; backoff on 429/503; randomise request intervals.
- All fetched data is stored locally; **no redistribution** of raw bhavcopy beyond the user's machine.
- Adjustments are applied on-read with a stored factor table — raw OHLCV is never overwritten, so we can re-derive adjustments if a corporate action is corrected later.
- **Point-in-time discipline**: every row is timestamped with `as_of` (when the fact was true) and `fetched_at` (when we learned it). Backtests query by `as_of`.

**What we explicitly don't do in v1:**
- No paid vendors (TrueData, GDFL, Refinitiv) — revisit only if NSE blocks us at scale.
- No intraday / tick data — daily bars only.
- No broker API for historical data as a product feed (per-user only, not redistributable).

**Cost to run v1 data pipeline: ₹0.**

## Acceptance criteria
1. First sync completes without user intervention on a stable connection.
2. After sync, pulling 6 months of daily data for any ticker is < 50 ms.
3. Adjusted close matches Zerodha chart values within 0.5% across a 20-ticker spot check spanning at least 3 split/bonus events.
4. Corrupt or missing days are detected and re-fetched on next sync.
5. A symbol delisted mid-history still has its older data intact, flagged "delisted".
6. Ticker ontology resolves TCS / TCS.NS / INE467B01029 / Screener `tcs` to the same entity.

## Out of scope
- Intraday (minute-level) data.
- Options / futures.
- BSE-only stocks (v1 focus is NSE).
- Derivatives expiries, OI data.

## Dependencies
- F1.4 (Local store).
