# F2.4 — `/fa <ticker>` Fundamental Snapshot

## One-liner
One command gives a full fundamental verdict — quality, valuation, growth — with peer context.

## User story
> As a long-term investor, I want to type `/fa TCS` and instantly know: is this a good company, is it cheap, is it growing?

## Expected behaviour (PM view)
- User runs `/fa TCS`.
- Output follows the universal verdict format with three sub-verdicts:
  ```
  🟢 HIGH-QUALITY  |  🟡 FULLY-VALUED  |  🟢 GROWTH-STABLE
  ```
- Three blocks follow, each with a 0–10 score and 3–4 key numbers:
  - **Quality** — ROE, ROCE, Debt/Equity, free cash flow.
  - **Valuation** — P/E (vs 5-yr avg, vs sector), P/B, EV/EBITDA.
  - **Growth** — 3-yr revenue CAGR, margin trend, recent quarterly delta.
- Every number has an arrow (▲ / ▼ / →) vs its 5-year average or sector median.
- **Sources** lists Screener.in (Tier 2) + company filings (Tier 1) when used.
- Freshness: fundamentals cached daily; user can force refresh with `--fresh`.
- If data is missing for a metric, show `—` rather than a fake value.
- Fundamentals are **normalised**: standalone vs consolidated is explicit, Ind-AS transitions handled, and sector-specific line items (banks/NBFCs/insurers use different P&L shapes) are mapped to a common schema where it makes sense — and kept separate where it doesn't.

## 🏰 Moat contribution
The second load-bearing moat feature alongside F2.1. A clean, normalised fundamentals corpus for Indian listed companies — with sector-aware schemas and point-in-time snapshots (no look-ahead) — is what separates Lakshmi from "LLM + Screener scrape". Point-in-time is non-negotiable: backtests and historical analysis must see only what was known on that date.

## Data strategy (v1)
**No paid APIs. No partnerships. Two-tier source strategy so we ship fast now and upgrade to pure Tier-1 later.**

| Dataset | v1 primary | v1.5 upgrade path |
|---|---|---|
| Quarterly & annual financials (P&L, BS, CF) | **Screener.in** scraped politely, cited as Tier 2 | Parse **NSE/BSE XBRL filings** ourselves (Tier 1, redistributable) |
| Key ratios (ROE, ROCE, D/E, margins) | Computed locally from Screener-provided line items | Computed from XBRL-parsed filings |
| Shareholding, promoter pledges | **BSE / NSE corporate filings** (Tier 1) | Same |
| Insider trades, SAST disclosures | **SEBI** public feeds (Tier 1) | Same |
| Concall transcripts, investor presentations | **Company IR pages** (Tier 1) — one-time crawl per quarter per covered company | Same, expanded coverage |
| Sector / industry classification | NSE industry master + BSE industry master (Tier 1) | Same |
| News headlines | **Moneycontrol / LiveMint / BusinessLine RSS** (Tier 3) | Curated feeds, same tier |

**Rules of the road:**
- Screener scraping respects `robots.txt`, rate-limits to ≤1 req/sec, caches for 24h, and never redistributes raw content. Cited as Tier 2 in every answer.
- Fundamentals are stored with `reported_at` (filing date) and `period_end` (fiscal period) — so point-in-time queries are trivial.
- Normalisation layer converts vendor-specific line items into a canonical schema (with sector variants for banks / NBFCs / insurers).
- If a number disagrees between Screener and the underlying filing, the **filing wins** and Screener is flagged stale.

**What we explicitly don't do in v1:**
- No licensed data deals (Screener, Trendlyne, Tijori) — revisit at paid-tier launch.
- No full-text news licensing — headlines + link only.
- No XBRL pipeline yet (it's the v1.5 moat-hardening project).

**Cost to run v1 fundamentals pipeline: ₹0.** The only trade-off is legal-grey Screener scraping; this is acceptable for free beta but must be replaced before a paid product tier.

## Acceptance criteria
1. Every metric shown has a source citation it can be traced to.
2. Numbers match the underlying source within 1% on spot checks.
3. Running `/fa` on a non-covered company (e.g. SME, very new listing) shows a graceful "limited data" message.
4. Full render in under 2 seconds from cache.
5. Three sub-verdicts never all contradict the detail numbers (internal consistency check).

## Out of scope

- Concall RAG (`/concall`) — v1.5.
- Sector-deep scorecards (banks vs manufacturing differ — v1.5).
- Editable assumptions.
