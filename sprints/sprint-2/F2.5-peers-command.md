# F2.5 — `/peers <ticker>` Peer Comparison

## One-liner
Rank a stock against its sector peers on the eight metrics that matter, in one table.

## User story
> As an investor deciding between similar stocks, I want to see TCS vs INFY vs WIPRO on a single screen with the numbers that matter, so I can compare apples to apples.

## Expected behaviour (PM view)
- `/peers TCS` shows a table with TCS and 4–6 nearest peers (same sector + comparable size):
  ```
  SYMBOL     P/E   P/B   ROE   ROCE   D/E   Rev 3y   NPM    MCap
  TCS       28x   12x   52%   63%   0.08    12%    19%    14.2 L
  INFY      25x   10x   31%   38%   0.10    10%    17%     6.8 L
  WIPRO     22x    6x   18%   22%   0.15     5%    13%     2.4 L
  …
  ```
- Each cell is colour-coded: best-in-peer green, worst red, middle neutral.
- A short verdict appears above the table:
  ```
  🟢 TCS ranks best on quality (ROE, ROCE); valuation is highest in peer set.
  ```
- User can change the peer set manually: `/peers TCS --vs INFY WIPRO HCLTECH`.
- Sort toggle: `/peers TCS --sort pe` reorders the table.
- Sources listed for the metrics (Tier 2 — Screener).

## Acceptance criteria
1. Default peer set is sensible for 90% of tickers (spot-check 20 across sectors).
2. The table never exceeds terminal width — extra columns collapse with a "more" hint.
3. A ticker with no clear peers (e.g. unique business) returns a clear message, not a random set.
4. Running twice in the same session is instant (cache).
5. Colour coding remains readable on both light and dark terminals.

## Out of scope
- Custom metrics / formulas.
- Relative valuation charts over time (v1.5).
- Cross-sector comparisons.

## Dependencies
- F2.4 (FA data pipeline already pulls the metrics).

## Demoable in
Run `/peers TCS`, `/peers HDFCBANK`, `/peers MARUTI` — show default peer sets + one manual override.
