# F2.7 — `/portfolio rebalance` Suggestions

## One-liner
Actionable, tax-aware trim/add suggestions that move the portfolio toward the user's target allocation.

## User story
> As an investor with a target allocation in mind, I want Lakshmi to look at my holdings and tell me exactly what to trim and what to add to get back on target — without me doing the math.

## Expected behaviour (PM view)
- Preconditions: user has a `~/.lakshmi/targets.yaml` with sector or asset allocation targets. If missing, Lakshmi offers to generate a sensible default and walks through it.
- User runs `/portfolio rebalance`.
- Output shows:
  - **Current allocation** vs target — horizontal bars with ⚠️ / ✓ indicators.
  - **Suggested actions** — numbered list of TRIM / ADD actions, each with:
    - Symbol
    - Rupee amount
    - One-sentence rationale (grounded: cites valuation / concentration / quality).
  - **Expected post-rebalance allocation** — shows the user how close they get.
  - **Tax impact warning** — STCG/LTCG estimate; flags if waiting N days saves money.
  - **Sources block** — Screener, holdings, user targets.
- A closing hint offers an export: `/export rebalance --to kite-basket`.
- **Never places orders.** Suggestions only.

## Acceptance criteria
1. Suggested actions, if executed in full, bring every sector to within 2% of its target.
2. No suggested TRIM exceeds the actual held quantity.
3. Tax warning correctly identifies STCG/LTCG based on cost basis + purchase date from Zerodha.
4. Rationale bullets each cite a source (no ungrounded "this looks pricey").
5. Running with missing targets triggers a friendly setup flow, not an error.

## Out of scope
- Executing the trades (v2, post-audit).
- Optimising for minimum transactions (simple greedy is fine for v1).
- Rebalancing across mutual funds / debt.
- Goal-based rebalancing (that's F3.5 — Yojanas).

## Dependencies
- F2.6 (Health) — shares the target-allocation concept.
- F2.4 (FA) — rationale needs fundamentals.
