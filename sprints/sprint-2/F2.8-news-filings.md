# F2.8 — News, Filings & Symbol Resolution Tools

## One-liner
Give the agent the tools it needs to actually answer "why" questions — filings, news, and a fuzzy symbol resolver.

## User story
> As a user typing "why is Hindustan Uniliver down?" (with a typo), I want Lakshmi to resolve the symbol, check for corporate announcements, scan the last few days of headlines, and tell me the most plausible reason — with sources I can verify.

## Expected behaviour (PM view)
Four new tools the planner (F2.0) can choose from:

### 1. `symbol_resolve(query)`
- Fuzzy-matches free text to an NSE symbol.
- "Hindustan Uniliver" → `HINDUNILVR` (typo tolerant).
- "hdfc" → disambiguation list (HDFCBANK, HDFCAMC, HDFCLIFE, …) with their full names.
- Runs on the ticker ontology from F2.1 (local, <10ms, no network).
- Tier: derived.

### 2. `quote(symbol)` and `historical(symbol, days)`
- `quote` — current LTP, day OHLC, % change, volume. Via Kite (broker tier) during market hours; falls back to last bhavcopy close off-hours.
- `historical` — N trailing days of adjusted OHLCV from the local store.
- Cache TTLs: quote 30s (live) / 5min (off-hours); historical until next market close.
- Tier: broker (Kite) or official (bhavcopy).

### 3. `corporate_announcements(symbol, days)`
- Last N days of BSE/NSE corporate filings for the symbol — results, dividends, splits, board meetings, insider trades, large deals.
- Each item has a headline, a filing date, and a URL back to the primary source.
- Tier: **official** (Tier 1). This is the highest-trust "why" source.

### 4. `news_search(query, days)` *(optional for first ship)*
- Indian-market news search via Tavily (or DuckDuckGo HTML fallback).
- Returns headline + publisher + timestamp + URL.
- Tier: derived (Tier 3). Lower trust than filings, but catches narrative ("Nomura downgrade", "management resignation") that filings miss.
- Off by default; enabled when the user sets `TAVILY_API_KEY` (BYO key, zero cost to us).

### Combined behaviour
- For "why is HUL down?" the planner typically runs: `symbol_resolve` → `quote` + `historical` + `index_quote(NIFTYFMCG)` + `corporate_announcements` (+ `news_search` if enabled).
- The rendered answer looks like:
  > 🔴 HUL is down 1.8% today and -4.2% over the past week [1][2].
  >  - FMCG index is roughly flat — the move is stock-specific [3].
  >  - Q4 results filed on 16 Apr showed volume growth of 2% vs 4% expected [4].
  >  - No other material filings in the past 30 days [4].
- Every number carries its source, and sources disclose their tier honestly.

## Acceptance criteria
1. `symbol_resolve("Hindustan Uniliver")` returns `HINDUNILVR` with confidence score; `symbol_resolve("xyz")` returns an empty list, not an error.
2. `quote(HINDUNILVR)` never returns a value older than 5 minutes during market hours, nor older than the last trading day off-hours — age is exposed in the tool's result and visible in the source's `fetched_at`.
3. `corporate_announcements(HINDUNILVR, 30)` returns only filings from the last 30 days, sorted newest first, with primary-source URLs.
4. `news_search` is gracefully absent when the API key isn't configured: the planner can still answer using quote + filings + historical.
5. Tool errors (rate limit, network down, BSE site change) surface as structured errors the planner can reason about (F2.0 AC#3), not empty results that look like success.
6. All four tools respect the F1.4 cache with sensible TTLs so a second `/ask` about the same symbol is cheap.

## 🏰 Moat contribution
Corporate announcements are Tier 1 and free. Every competitor (AI assistants, Twitter, news aggregators) is reading the *same* feed — but most strip the URL, paraphrase the headline, or bury it under ads. Lakshmi cites it verbatim, with a clickable link back to BSE. That alignment between *what we show* and *what the regulator published* is a correctness moat — small per query, compounding across thousands.

## Out of scope
- Macro / economic news (RBI policy, inflation prints) — v2.
- Insider trade pattern detection — just surface filings in v1; pattern analysis comes in a later sprint.
- Sentiment scoring — the user reads the headline, not a summary.
- Paid news vendors (Reuters, Bloomberg) — explicitly rejected per Sprint 2 data strategy.

## Dependencies
- F2.0 (Agentic planner — otherwise these tools can't be composed).
- F2.1 (NSE pipeline — `historical` reads from the local store; `symbol_resolve` reads the ontology).
- F1.4 (Local store — cache layer).

## Demoable in
Ask "why is HUL down?" with the Tavily key unset, then with it set. Point out that the core answer (filings + price context) is there either way; news is a bonus, not a dependency.
