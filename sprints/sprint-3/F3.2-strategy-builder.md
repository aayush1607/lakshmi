# F3.2 — Conversational Strategy Builder + Mini Backtest

## One-liner
User describes a strategy in plain English; Lakshmi clarifies, generates a Mantra, backtests it, and shows the results.

## User story
> As a retail investor curious about systematic strategies, I want to describe a momentum idea in natural language and have Lakshmi turn it into a backtested, saveable playbook — without writing any code.

## Expected behaviour (PM view)
- User says: `build me a momentum strategy for mid-caps`.
- Lakshmi asks a short set of clarifying questions with sensible defaults the user can accept or tweak:
  1. Universe (default: NIFTY Midcap 150)
  2. Momentum measure (default: 12-1 month return)
  3. Rebalance frequency (default: monthly)
  4. Position count + weighting (default: top 15 equal-weight)
  5. Liquidity filter (default: min ₹500 Cr avg daily volume)
- User types `accept` (or overrides any field).
- Lakshmi runs a backtest over the available NSE history (F2.1) and shows:
  ```
  CAGR, Max DD, Sharpe, Hit rate, Turnover
  ```
  compared against the benchmark index.
- A **Monte Carlo** block shows P5 / P50 / P95 CAGR bands from bootstrap resampling.
- Strategy is saved as `~/.lakshmi/mantras/<name>.yaml` and can be run via `/mantra run <name> --paper`.
- User can edit the YAML and re-backtest with `/mantra backtest <name>`.

## Acceptance criteria
1. A user can go from typing the initial request to a saved, backtested mantra in under 3 minutes.
2. Backtest results are reproducible — running the same mantra twice gives identical numbers.
3. Backtest respects the liquidity filter (no illiquid names in positions).
4. Costs and slippage use transparent, stated defaults (e.g. 20 bps per trade).
5. The generated YAML is human-readable and editable.

## Out of scope
- Live / real-money execution.
- Short selling, options, intraday.
- Multi-factor optimisation / walk-forward (keep v1 simple).
- Strategy marketplace.

## Dependencies
- F3.1 (Mantra engine)
- F2.1 (NSE data history)

## Demoable in
Recording: describe the idea, walk through clarifications, accept defaults, watch the backtest run, view saved YAML, run again with an edited parameter.
