# F3.3 — Darshans (Live Dashboards)

## One-liner
Three one-screen live dashboards: my portfolio, a single equity, the whole market.

## User story
> As a user who wants a live pulse without juggling tabs, I want Lakshmi to render a single auto-refreshing screen that tells me everything I need to know right now.

## Expected behaviour (PM view)
Three dashboards, each one-screen and auto-refreshing every 5 seconds during market hours:

### `darshan portfolio`
- Top: total P&L, day P&L, invested / current value.
- Middle: holdings table with live LTP, day change, contribution to day P&L.
- Bottom: sector allocation bar + biggest mover of the day.

### `darshan equity <ticker>`
- Header: price, day change, volume vs avg.
- Left half: mini 3-month chart + key levels.
- Right half: key fundamentals (P/E, ROE, growth) + today's news headlines.
- Footer: verdict row from `/ta` + verdict row from `/fa`.

### `darshan market`
- NIFTY, BANKNIFTY, NIFTY Midcap — LTP + day change + mini chart.
- Advance/decline ratio.
- FII/DII cash flows (prior day).
- India VIX.
- Top 5 gainers / losers / volume leaders.

Common behaviour:
- `q` or `Esc` exits.
- `r` forces refresh.
- Everything is grounded — every number has a source tier; sources are shown on `s`.
- If data stream pauses (market closed), header flips to "post-close snapshot".

## Acceptance criteria
1. Each dashboard fits a single 120×40 terminal with nothing cut off.
2. Refresh does not flicker — partial updates only.
3. Keyboard shortcuts (`q`, `r`, `s`) respond within 100 ms.
4. Works offline with last-known values and a clear "stale — N min old" banner.
5. Opening and closing a darshan 10 times in a row shows no memory growth.

## Out of scope
- Custom / user-configurable darshans.
- Multi-monitor / tmux-style layouts.
- Streaming tick data (5s polling is enough).
- News article full-text (headlines only).

## Dependencies
- F1.3, F2.1, F2.2, F2.4 — darshans reuse these engines.
