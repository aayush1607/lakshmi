# F3.4 — Aartis (Alerts)

## One-liner
YAML-defined conditional alerts that run in the background and notify the user when triggered.

## User story
> As an investor watching for specific setups, I want to set conditions like "tell me when TCS RSI > 70 and P/E > 28" and have Lakshmi ping me without me checking every day.

## Expected behaviour (PM view)
- User creates an alert interactively: `/aarti new` — Lakshmi asks for name, conditions, channels.
- Or, user drops a YAML into `~/.lakshmi/aartis/`:
  ```yaml
  name: "TCS trim signal"
  when:
    - symbol: TCS
      pe: "> 28"
      rsi_14: "> 70"
  notify:
    - terminal
    - telegram
  cooldown: 24h
  ```
- A background evaluator (started by `lakshmi` or `lakshmi daemon start`) checks conditions once per minute during market hours, once per day for fundamentals.
- When a condition fires:
  - In-terminal toast when Lakshmi is running.
  - System notification (macOS / Linux).
  - Optional Telegram message if configured.
- `/aarti list` shows all defined alerts with their last-fired timestamp and state.
- `/aarti mute <name>` silences without deleting.
- `/aarti test <name>` evaluates it right now, regardless of cooldown, for debugging.
- Cooldown prevents re-firing the same alert within the specified window.

## Acceptance criteria
1. An alert created via YAML is picked up within 60 seconds — no restart needed.
2. Conditions support price, volume, RSI, P/E, P/B, day-change, % from 52-week high/low.
3. Telegram notifications include a link / snippet grounded in the same sources Lakshmi shows in-app.
4. When Lakshmi isn't running, the optional daemon still evaluates alerts.
5. A malformed YAML surfaces a clear, line-numbered error in `/aarti list`.

## Out of scope
- Complex boolean trees (AND is supported; OR via multiple alerts).
- Email / SMS / webhook integrations beyond Telegram (v1.5).
- Multi-leg / pair alerts.
- Alert history browser UI.

## Dependencies
- F2.1 (TA data) for RSI / volume conditions.
- F2.4 (FA data) for P/E etc.

## Demoable in
Create a simple "price above X" alert on a ticker currently near X. Wait for it to fire. Show the terminal toast + system notification.
