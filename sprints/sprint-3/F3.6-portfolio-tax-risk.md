# F3.6 — Portfolio Tax / Risk / Attribution Lenses

## One-liner
Three additional portfolio views — `/portfolio tax`, `/portfolio risk`, `/portfolio attribute` — that turn Lakshmi from a snapshot into a proper analytics suite.

## User story
> As an investor, I want to know **why** my portfolio moved today, **what** it'll cost me in tax if I rebalance, and **how much risk** I'm actually carrying — in one command each.

## Expected behaviour (PM view)

### `/portfolio attribute`
- Answers "why is my portfolio up/down today?"
- Shows the top 5 positive and top 5 negative contributors to today's P&L (in rupees and in percentage points of portfolio return).
- One-line explanation per contributor if news is available ("+₹4,200 TCS: Q4 results beat estimates").
- Works for `--today` (default), `--week`, `--month`, `--custom from to`.

### `/portfolio tax`
- Shows realised and unrealised gains broken into STCG vs LTCG pockets.
- Highlights **harvesting opportunities**:
  - Losses that can offset existing gains this FY.
  - Positions nearing LTCG threshold — "N days to LTCG eligibility saves ₹X".
- FY-aware — defaults to current Indian FY (Apr–Mar); user can pick another.

### `/portfolio risk`
- Portfolio beta vs NIFTY 50.
- Concentration index (top-5 share, HHI).
- Max drawdown over the last 1y / 3y / 5y using actual historical holdings (approximation: current weights applied to history).
- Sector clustering — "your portfolio behaves like 0.82 correlated to NIFTY IT".
- Single-factor vulnerability ("85% sensitivity to USD/INR via IT exposure").

**Common:**
- All three follow the universal verdict format: a one-line take, then the detail.
- All are grounded — holdings from Zerodha, prices from NSE, fundamentals from Screener.
- All cite their sources.

## 🏰 Moat contribution
`/portfolio tax` is where India-specific correctness becomes defensible. Indian FY (Apr–Mar), 12-month LTCG rule for equities, section 112A grandfathering (31-Jan-2018 basis), STT treatment, buyback tax — these are easy to get subtly wrong, and wrong answers here cost users real money. A generic LLM wrapper will not encode this reliably. Treat the tax rule set as a unit-tested, versioned module, not a prompt.

## Acceptance criteria
1. `/portfolio attribute --today` sum of contributions equals the day P&L in F1.3 within ±₹1 rounding.
2. `/portfolio tax` STCG/LTCG classification is correct vs Indian tax rules (holding period > 12 months = LTCG for equities).
3. `/portfolio risk` beta values are within 0.05 of values computed by hand on the same data.
4. Each command runs in under 2 seconds from cache.
5. Commands degrade gracefully for portfolios with < 1 year of history (say "insufficient history for risk metrics").

## Out of scope
- Filing-ready tax reports / P&L statement export (v1.5).
- Advance tax estimation.
- Factor attribution (Fama-French, etc.).
- VaR / CVaR, tail-risk stats.

## Dependencies
- F1.3 (portfolio), F2.1 (prices), F2.4 (fundamentals), F1.6 (response shaper).

## Demoable in
Run all three on a real portfolio. Highlight one insight from each that the user couldn't easily have gotten elsewhere (e.g. "you're 0.91 correlated to IT — more than you think").
